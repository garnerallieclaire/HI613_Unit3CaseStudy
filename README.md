# HI613 Unit 3 Case Study – CPOE Functional Modeling

## Overview
This project expands on prior requirements analysis by developing functional models for a Computerized Physician Order Entry (CPOE) system. The project includes use case diagrams, activity diagrams, and detailed use case descriptions to represent clinical workflows for medication ordering, validation, processing, and administration.

## Objectives
- Translate system requirements into functional models
- Visualize clinical workflows using diagrams
- Ensure consistency between requirements and system behavior
- Model interactions between clinical roles and system components

## Tools Used
- Lucidchart (diagram creation)
- Systems analysis and UML modeling techniques
- Microsoft Word (documentation)

## Project Summary

### Functional Modeling
- Developed a comprehensive use case diagram showing interactions between:
  - Physician  
  - Pharmacist  
  - Pharmacy Technician  
  - Nurse  
  - Clinical Decision Support (CDS) system  

- Created activity diagrams for key workflows:
  - Medication ordering  
  - Medication verification (including CDS checks)  
  - Medication processing in the pharmacy  
  - Medication administration by nursing staff  

### Use Case Development
- Defined detailed use case descriptions for:
  - Order Medication  
  - Verify Medication Order  
  - Process Medication Order  
  - Administer Medication  

- Included:
  - Primary actors and stakeholders  
  - Triggers and normal workflows  
  - Subflows and exception handling  

### Clinical Workflow Integration
- Incorporated decision support logic:
  - Dose validation  
  - Allergy and drug interaction checks  
  - Inventory availability  

- Modeled real-world healthcare processes:
  - Multi-step medication validation  
  - Role-based responsibilities across care teams  
  - Documentation of administration, waste, and patient response  

### Requirements Refinement
- Updated requirements from previous case study to reflect:
  - Improved workflow structure (separating ordering and verification)  
  - Integration of CDS system as a key component  
  - Additional clinical documentation requirements (e.g., patient refusal)   

## Key Skills Demonstrated
- Functional modeling (UML diagrams, activity diagrams)
- Healthcare workflow analysis
- Clinical decision support integration
- Systems analysis and design
- Translating requirements into system behavior
- Understanding of interdisciplinary clinical processes
